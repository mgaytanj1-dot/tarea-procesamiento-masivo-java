Descripción del problema:
Al procesar una masiva cantidad de clientes, el programa consume muchos recursos principalmente en la ram y volvia el proceso mucho mas lento

Versión ineficiente:
cargaba todos los clientes en memoria y utiliza listas con búsquedas lineales para agrupar campañas, lo que provoca un rendimiento más lento.

Versión optimizada:
Se utilizo HashMap para agrupar campañas de manera más rápida y eficiente, mejorando el tiempo de procesamiento y el manejo de memoria.

conclusión personal:
A pesar de que un programa funcione, se deben usar las estructuras de datos correctas paraque funcione de manera perfecta sin consumir tantos recursos
